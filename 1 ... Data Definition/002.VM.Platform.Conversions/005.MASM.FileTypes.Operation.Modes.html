File Types Created by MASM (Microsoft Macro Assembler)

MASM is a powerful assembler that supports different operating modes and generates various file types, depending on the architecture you're targeting. Understanding these modes is key to knowing what kind of assembly code you're writing and how it interacts with the operating system.

32-Bit Protected Mode

  • Supported by all 32-bit and 64-bit Windows operating system versions. This is the standard for modern Windows applications.
  • Generally easier to write and debug compared to the older 16-bit real mode, as it offers memory protection and a larger address space.
  • This is the default mode that most MASM tutorials, including many parts of your book, will focus on.

64-Bit Long Mode

  • This is the native mode for all 64-bit versions of Windows and other modern 64-bit operating systems.
  • Requires the use of different registers (like RAX, RBX, RCX, RDX instead of EAX, EBX, etc.) and updated calling conventions for functions.
  • To work with 64-bit assembly, you'll need specific tools like MASM64 (part of the Windows SDK) or other x64 assembly tools.

16-Bit Real-Address Mode

  • This is a legacy mode primarily used in older operating systems like MS-DOS and for programming embedded systems or BIOS routines.
  • It's characterized by direct memory access without protection, and a smaller 1MB address space.
  • Not directly supported on modern 64-bit Windows operating systems without using emulators (like DOSBox) or virtual machines.
