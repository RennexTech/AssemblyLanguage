Ever wondered how computers store and interpret numbers and characters? It's like having different ways to write down the same idea! From the raw binary bytes they munch on to the human-friendly symbols we use, understanding these forms is key to speaking fluent computer. Let's break it down!

Term Meaning
Binary Integer A number stored in raw binary, the language of computers. It's used for actual calculations. For example, the decimal number 65 is stored as `01000001` in an 8-bit binary integer. Think of it as the CPU's favorite snack!
Digit String A sequence of ASCII characters, where each character happens to be a digit. So, the number "123" isn't a single numerical value to the computer; it's three separate characters: '1' (`31h`), '2' (`32h`), and '3' (`33h`). This takes up more space than a binary integer but is super easy for humans to read and display.
ASCII Character A fundamental building block of text! Each symbol you seeâ€”like `'A'`, `'0'`, `'$'`, or `' '` (space)â€”has a unique ASCII code, which is a numerical value (usually 0-127) stored in memory. When you type 'A', the computer stores `41h` (hexadecimal) or `65` (decimal). It's how text becomes digital.
Hexadecimal A base-16 number system that's a super convenient shorthand for binary. It uses digits 0-9 and letters A-F. Each hexadecimal digit represents exactly 4 binary bits, making it much easier for humans to read and write large binary numbers. For example, `41h` is `65` in decimal, and that's ASCII for `'A'`. It's like a compressed zip file for binary data!
Octal A base-8 number system, using digits 0-7. Each octal digit represents 3 binary bits. While less common than hexadecimal today, it's still super useful in specific contexts, especially in Unix-like systems for setting file permissions (e.g., `755o` for read/write/execute permissions). It's like an old-school, but still effective, tool in the hacker's toolkit.
