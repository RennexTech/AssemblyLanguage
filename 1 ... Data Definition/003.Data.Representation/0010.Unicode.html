Character encodings are crucial for computers to understand and display text from different languages around the world. Here's a breakdown of the most common Unicode Transformation Formats (UTFs), explaining how they store characters.

Format Description Size Common Use Cases
UTF-8 A variable-width encoding. This means characters can take up different amounts of space. It's designed to be backward-compatible with ASCII, meaning ASCII characters (0-127) are represented using a single byte, just like in ASCII. This makes it very efficient for English text. UTF-16 A variable-width encoding, but characters are stored in units of 16 bits (2 bytes). Most common characters fit into a single 2-byte unit. Less common characters (like some emojis or historical scripts) require two 2-byte units (4 bytes total), using a mechanism called "surrogate pairs." 2 or 4 bytes
  • 2 bytes for most characters (Basic Multilingual Plane)
  • 4 bytes for supplementary characters (using surrogate pairs)
 Primarily used in Windows operating systems and their APIs, Java's internal string representation, and JavaScript's internal string representation.
UTF-32 A fixed-width encoding. Every single Unicode character, regardless of its complexity or language, is represented using exactly 32 bits (4 bytes). This makes it very simple to work with (e.g., finding the Nth character is easy), but it's highly inefficient for storage and transmission, especially for text that primarily uses common characters.
